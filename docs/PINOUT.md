## TFT 屏幕连接 (1.69 寸 8 针 SPI)

### 实物接线图

```
                    ┌─────────────────────────────────────┐
                    │      ESP32-S3-N16R8 开发板          │
                    │                                     │
         3.3V ──────┤● 3V3     IO45 ●─────────────────────┤
         GND ───────┤● GND     IO0  ●─────────────────────┤
       IO12 ────────┤● IO12    IO46 ●─────────────────────┤
       IO11 ────────┤● IO11    IO9  ●─────────────────────┤
       IO10 ────────┤● IO10    IO8  ●─────────────────────┤
       IO9  ────────┤● IO9     IO47 ●─────────────────────┤
       IO15 ────────┤● IO15    IO48 ●─────────────────────┤
       IO14 ────────┤● IO14    IO44 ●─────────────────────┤
                    │                                     │
                    │            USB-C 接口               │
                    └─────────────────────────────────────┘
                       ▲    ▲    ▲    ▲    ▲    ▲    ▲
                       │    │    │    │    │    │    │
                       │    │    │    │    │    │    └──→ IO14 ──→ BLK (背光)
                       │    │    │    │    │    └───────→ IO15 ──→ RES (复位)
                       │    │    │    │    └────────────→ IO9  ──→ DC (数据/命令)
                       │    │    │    └─────────────────→ IO10 ──→ CS (片选)
                       │    │    └──────────────────────→ IO11 ──→ SDA (MOSI)
                       │    └───────────────────────────→ IO12 ──→ CLK (SCK)
                       └────────────────────────────────→ 3.3V ──→ VCC
```

### 表格形式

| TFT 引脚 | ESP32-S3 GPIO | 功能说明 | 线色建议 |
|----------|---------------|----------|----------|
| VCC | 3.3V | 电源正极 | 红色 |
| GND | GND | 电源地 | 黑色 |
| CLK/SCK | GPIO12 | SPI 时钟 | 绿色 |
| SDA/MOSI | GPIO11 | SPI 数据输出 | 黄色 |
| CS | GPIO10 | 片选信号 | 橙色 |
| DC/RS | GPIO9 | 数据/命令选择 | 蓝色 |
| RES/RST | GPIO15 | 复位信号 | 紫色 |
| BLK/BLED | GPIO14 | 背光 PWM 控制 | 白色 |

## 按钮连接 (可选)

### BOOT 按钮 (进入配网模式)

```
BOOT 按钮
    │
    ├───→ GPIO0 (BOOT 引脚)
    │
    └───→ GND (通过按钮按下接地)

ESP32-S3 内部已有上拉电阻，无需外接
```

**使用方法：**
1. 按住 BOOT 键
2. 按下 RST 键
3. 继续按住 BOOT 键 3 秒
4. 设备进入 AP 配置模式

### RST 按钮

```
RST 按钮
    │
    ├───→ EN (使能引脚)
    │
    └───→ GND (通过按钮按下接地)

开发板上通常已有此按钮
```

## 电源连接

### USB 供电 (推荐)

```
USB-C 接口 ──→ 开发板 ──→ 3.3V LDO ──→ 各组件
   5V/2A                    │
                           ├──→ TFT 屏幕 (VCC)
                           ├──→ ESP32-S3 (核心)
                           └──→ 其他外设
```

### 外部电源 (可选)

```
5V 电源 ──→ 5V 引脚
            │
            └──→ 内部 5V 网络

3.3V 电源 ──→ 3V3 引脚
              │
              └──→ 内部 3.3V 网络 (绕过 LDO)
```

## 接线注意事项

### ⚠️ 警告

1. **不要使用 5V 信号连接 TFT 数据引脚**
   - TFT 屏幕使用 3.3V 逻辑电平
   - 5V 可能会损坏屏幕驱动 IC

2. **电源电流要求**
   - 静态电流：~100mA
   - 峰值电流（WiFi 发射 + 屏幕背光）：~500mA
   - 建议使用 5V/2A USB 电源

3. **GPIO 引脚选择**
   - 某些 GPIO 有特殊用途（如 strapping pins）
   - 推荐使用表中指定的引脚

### ✅ 推荐

- 使用杜邦线连接，方便调试
- 热缩管或电工胶带包裹裸露部分
- 焊接后检查是否有短路
- 先接电源，测量电压正常后再连接数据引脚

## 测试步骤

### 1. 通电测试

```bash
# 连接 USB 线，测量各点电压
3.3V 引脚 → 应为 3.3V ±0.1V
5V 引脚  → 应为 5.0V ±0.2V
```

### 2. 串口连接

```bash
# 连接 USB 后，运行：
idf.py monitor

# 应该能看到启动信息
```

### 3. 屏幕测试

如果接线正确，设备启动后屏幕会显示：
- 启动画面
- WiFi 状态
- AI 状态

### 4. 按钮测试

```bash
# 按住 BOOT 键，按 RST，打开串口监控
# 应该能看到进入 AP 模式的提示
```

## 故障排除

### 屏幕不亮

1. 检查 VCC 和 GND 接线
2. 测量供电电压是否为 3.3V
3. 检查 BLK 引脚是否有 PWM 信号
4. 调整亮度：`nanobot> display brightness 100`

### 屏幕显示乱码

1. 检查 CLK、SDA、CS、DC 接线是否正确
2. 检查是否使用了正确的 GPIO 引脚
3. 确认 SPI 频率是否过高（默认 40MHz）

### 无法进入配网模式

1. 确认 BOOT 键连接到 GPIO0
2. 按住 BOOT 键 3 秒以上
3. 查看串口输出确认
